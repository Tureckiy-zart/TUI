# Select

Foundation Select is a **LOCKED** dropdown selection primitive built on **Radix Select**. It provides a stable compound API and token-driven styling, while delegating accessibility and keyboard behavior to Radix.

## Purpose

- Provide a single canonical dropdown selection primitive for the entire UI system.
- Ensure consistent styling via token unions and consistent behavior via Radix.

## When to use

- Use for single-value selection from a list of options.
- Use inside forms (with a real label) and in UI controls where a listbox/combobox interaction is appropriate.

## When NOT to use

- Do not use for multi-selection (use an Extension that composes Select internally).
- Do not use for free-form input (use `Input`).
- Do not reimplement Select behavior (custom focus/keyboard/ARIA) â€” Select is the foundation primitive.
- Do not use `LegacySelect` or `selectVariants` (HTML `<select>` legacy implementation). It is backward-compatibility-only and forbidden for new usage.

## Behavioral guarantees

- **Single canonical focus flow**: focus moves to the trigger and returns predictably after selection/close (Radix-managed).
- **Keyboard navigation**: Arrow keys, Enter/Space, Escape behave according to Radix Select semantics.
- **Controlled vs uncontrolled**:
  - Controlled: `value` + `onValueChange`
  - Uncontrolled: `defaultValue`
- **No layout responsibility**: layout is owned by the parent; the component exposes token width on the trigger, not external layout rules.
- **Token-only visuals**: size/variant/width/offsets are token-based. No raw visual values are part of the public API.

## Locked rationale

Select is part of the **Foundation (LOCKED)** layer. Any change to behavior, public exports, or token domains requires the official **UNLOCK** procedure and an audit. The only allowed changes inside the locked component are bug fixes, typing fixes, and documentation/story updates that preserve the public API contract.

## Props

`Select` is a compound namespace. Props are defined on subcomponents (`Select.Root`, `Select.Trigger`, `Select.Content`, `Select.Item`, etc.), not on `Select` itself.
